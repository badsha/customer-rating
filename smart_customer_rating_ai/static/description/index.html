Why Teams Use This

Multi-Template Assessments

Maintain multiple assessment templates per customer (e.g., Sales fit, Risk, Support). Each template stays consistent across all customers.

Master Assessment Sync

When criteria change in the master assessment, customer ratings stay aligned automatically.

Fast Visual Priority

Tag badge and stars provide quick risk/opportunity scanning in list and form views.

Automated Rating + AI Insights

Compute an automated rating from configurable rules, including optional AI sentiment analysis (Ollama / OpenAI / Gemini) with a transparent rule breakdown.

Key Features

  • Multiple assessment templates with synchronized criteria lines.
  • Manual scoring per template (stars) with automatic average score and star rendering.
  • Tag classification (Watchlist, Rising Star, Champion) for quick segmentation.
  • Change timeline with who changed, when changed, and score impact.
  • Automated rating engine with configurable rules and "Rule Breakdown" (measured value, points, threshold met).
  • Optional AI sentiment analysis (Ollama / OpenAI / Gemini) with provider visibility in insights.
  • AI Advisory Mode: Off, Advisory-only, or Blend AI scores into final rating.
  • Daily recompute scheduler for automated ratings (optional).
  • Bridge-friendly design for Contacts, CRM, Sales, and other apps.

AI Integration Setup

🤖 Local Ollama Setup (Recommended)

For local AI processing with Ollama:

1. Install Ollama:
curl -fsSL https://ollama.ai/install.sh | sh

2. Pull Model:
ollama pull llama3

⚠️ Important for Docker Users:

If Odoo runs in Docker, Ollama must listen on all interfaces:

OLLAMA_HOST=0.0.0.0 ollama serve

Then use your host IP in Odoo settings (e.g., http://192.168.1.100:11434)

3. Configure in Odoo:
Settings → Customer Rating → AI Intelligence
• AI Provider: Local Ollama
• Ollama URL: http://localhost:11434 (or host IP for Docker)
• Model: llama3

☁️ Cloud AI Options

For cloud-based AI processing:

  • OpenAI: Requires API key, uses GPT-4o model
  • Google Gemini: Requires API key, uses Gemini Pro model
  • Local Ollama: No API costs, runs locally, requires Ollama setup

💡 Pro Tip: Start with Local Ollama for testing, then choose cloud provider for production scaling.
